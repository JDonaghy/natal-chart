Validate swisseph-wasm first, before anything else. It's the only dependency you can't swap out, and getting ephemeris .se1 data files loading correctly in Vite is where projects stall. A 30-line spike that prints the Sun's position for a known birthday will tell you everything.
The Cloudflare Worker is simpler than it sounds. Even without a Cloudflare account yet, you can develop and test the geocoding worker locally with wrangler dev. You'll need the account only when you deploy.
Hash routing (#/) for v1. GitHub Pages + client-side routing is a classic footgun. The createHashRouter path in React Router is ugly but painless. You can migrate to proper paths later if you add a custom domain.
When you're ready to start building, a good first prompt for Claude Code would be something like: "Set up a pnpm monorepo with packages/core, packages/web (Vite + React + TypeScript), and packages/worker. Then spike swisseph-wasm in packages/core: load the library, compute Sun position for 1990-06-15 12:00 UTC at 51.5°N 0.1°W, and print the result."
